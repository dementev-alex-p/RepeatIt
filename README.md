# Repeat It
https://t.me/repeat_it_tgbot  

Telegram бот для изучения и долгосрочного запоминания информации. 
Пользователь вводит информацию, которую хочет выучить и запомнить в виде карточек (вопрос и ответ).
Бот проводит занятие, фиксируя результат запоминания по каждой карточке и на основе этой информации формирует план повторений.
Бот основан на алгоритме интервального повторения SM-2 (научно доказанный подход). 
Любой желающий может воспользоваться ботом в Telegram - @repeat_it_tgbot

## Что внутри
Важно! Данный репозиторий не создан для демонстрации скиллов автора, тут нет технологий ради технологий, интеграций ради интеграций и тд.
Это творческий "петпроджект для души", который решает конкретную проблему и реализован в свободное время
- Java 17
- Spring Boot 3.5.3
- PostgreSQL 13
- Hibernate
- Flyway
- Docker + docker-compose

## Основная функциональность
- Ежедневные тренировки - бот анализирует карточки пользователя и сам определяет что ему необходимо повторить сегодня
- Ручное создание карточки
- Генерация карточек с помощью GigaChat (интеграция с API Сбера)
- Импорт карточек в формате JSON
- Группировка карточек в коллекции
- Добавление карточек из публичных коллекций
- Изучение карточке коллекций
- Исключение коллекции из ежедневных тренировок
- Взаимодействие с пользователем через Single‑message паттерн - новое сообщение заменяет предыдущее, создавая иллюзию UI экрана в рамках чата


## Как запустить
0. Создать бота в BotFather и аккаует в developers.sber.ru для интеграции с GigaCat (опционально)
1. Склонировать репозиторий
2. Создать файл .env в корне:
```
DB_PASSWORD=ваш_пароль  
TG_BOT_TOKEN=токен_от_BotFather  
GIGACHAT_AUTH_KEY=ключ_из_developers.sber.ru
```
3. docker compose up -d

Контейнеры:
- postgres (порт 5432 на localhost)
- бот (порт 8080 и 5005 для отладки тоже только localhost)

Логи: docker logs -f repeat_it_bot
