# Базовая информация 

Тренировки основаны на системе интервального повторения, под капотом заложен алгоритме SM-2.
Основен тезисы системы интервального повторения 
- память ослабевает с течением времени, но каждое последующее повторение материала укрепляет ее и замедляет скорость забывания
- что бы всегда помнить информацию ее нужно повторить через 
  - 1 день
  - 3 дня 
  - 6 дней 
  - 17 дней
  - 50 дней
  - 150 дней
  - и тд
- приведенные выше интервалы актуальны для информации, которую пользователь с легкостью вспоминает
- если при очередной итерации информация забылась, то прогресс по карточке обнуляется 
- если при очередной итерации информация вспоминается с трудом, то следующий интервал будет короче

Ключевой показатель карточки - это EF (Easiness Factor - фактор легкости)
- по умолчанию он 2,5
- каждая итерация изменяет его по формуле  
  - EF' = EF + (0.1 - (5 - quality) * (0.08 + (5 - quality) * 0.02))
- где quality - это качество ответа пользователя 
  - 0 - забыл
  - 3 - вспомнил, но с трудом
  - 5 - легко вспомнил
- минимальное значение 1,3 
- после каждой итерации высчитывается дата следующего повторения, 
для этого к текущей интервал (I) карточки, он в свою очередь вычисляется 
в зависимости от количества подряд правильных ответов (streak)
  - streak = 0 => 1 день
  - streak = 1 => 3 дня
  - streak = 2 => 6 дней
  - streak > 3 => считается по формуле I' = I * EF (I - интервал)

# Как выглядит тренировка?
### Начало тренировки
- Пользователь нажимает на кнопку "Тренировка"
- Система формирует план на тренировку 
  - выгружаются карточки, у которых дата повторения в прошлом
  - сортируются по дате повторения 
  - создаются сущности в бд 
    - practice - сама тренировка, фиксируется время начала и текущая карточка 
    - practice_card - план тренировки, фиксируется какие карточки будут участвовать в тренировки и в какой последовательности 
- пользователю отображается информация о начале тренировки с количеством карточек и первая карточка 

### Ход тренировки
- Пользователю отображается карточка 
- Если карточка подразумевает лицевую и обратную сторону, то обратная сторона заблюрена
- Пользователь пытается вспомнить информацию на карточке 
- Пользователь выбирает один из вариантов результативности 
  - Не вспомнил (q=1)
  - Вспомнил с трудом (q=3)
  - Легко вспомнил (q=5)
- Система фиксирует результат и высчитывает прогресс по карточке
  - вычисляется фактор легкости (EF) и дата следующего повторения 
  - результат сохраняется в таблицу practice_card
- Пользователю отображается следующая карточка 

### Завершение тренировки 
- При определении следующей карточки система понимает, что тренировка завершена
- Система формирует статистику и сохраняет ее в таблице practice 
  - продолжительность тренировки 
  - количество и процент успешно запомненных карточек
- Пользователь видит статистику тренировки

Если пользователь не закончил тренировку 
- Система напомнит пользователю в 19:00 по его времени о необходимости продолжить тренировку 
- Если пользователь не продолжил тренировку до конца дня, то система завершит ее со статусом прервана и также посчитает статистику 

# Добавление карточек

Для создания карточек пользователь нажимает - "добавить карточку" в главном меню.
Система предлагает ему на выбор три действия: создать карточку, добавить коллекцию, импортировать карточки 

### Создание карточки 
Пользовательский сценарий:
1. Пользователь нажимает создать карточку
2. Система предлагает ввести лицевую сторону карточки
3. Пользователь вводит лицевую сторону
4. Система 
   1. создает карточку в бд
   2. предлагает ввести Обратную сторону карточки или нажать "Пропустить шаг"
5. Пользователь вводит обратную сторону или пропускает шаг
6. Система 
  - обновляет карточку в бд, если требуется 
  - выводит пользователю сообщение "Карточка успешно создана!" и демонстрирует карточку.
  - предлагает пользователю действия 
    - "Изменить карточку" (переход к п.2 только уже без создания в бд) 
    - "Добавить карточку в коллекцию"
    - "Создать еще одну"
    - "Вернуться в главное меню"

### Добавление карточки в коллекцию 
1. После создания карточки пользователь нажал "Добавить карточку в коллекцию"
2. Система 
   1. находит все его коллекции
   2. выводит сообщение "Выберите коллекцию" отображает коллекции в порядке последнего обновления
   3. каждой коллекции соответствует кнопка для выбора и отображается кнопка "Создать коллекцию"
3. Пользователь выбирает из списка 
4. Система проставляет связь между карточки и коллекции и выводит текст "Карточка добавлена в коллекцию"
5. Пользователь выбрал "Создать коллекцию"
6. Система предлагает ввести название коллекции 
7. Пользователь вводит название 
8. Система создает коллекцию и автоматически добавляет карточку в коллекцию
